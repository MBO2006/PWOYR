# PowerDoc-Agent 项目文件汇总（零基础版）

> 这个项目是干什么的？  
> 你有一份 Word 报告（比如"北流市××接入系统报告.docx"），里面有很多**可变的信息**（比如地名"北流市"、公司名"广西电网"、数字"300MW"）。  
> 你想做一个"模板"，把"北流市"变成 `{{A}}`，"广西电网"变成 `{{F1}}`，这样下次换一个城市（比如"桂林市"），只要在模板里填新值就能自动生成新报告。  
> 这个项目就是在干这件事——**解析 Word 文件，找到那些可变的内容，替换成占位符，生成模板**。

## 0. 你需要先知道的背景知识

<a id="s0-1"></a>
### 0.1 .docx 文件其实是一个 ZIP 压缩包

你可以在电脑上试一下：把任意一个 `.docx` 文件改成 `.zip`，然后双击打开，你会看到里面有很多文件夹和 XML 文件。

```
报告.docx → 重命名为 报告.zip → 打开看到：
  ├── word/
  │   ├── document.xml    ← 文档正文（最重要的）
  │   ├── header1.xml     ← 页眉（每一页顶部的内容）
  │   ├── footer1.xml     ← 页脚（每一页底部的内容）
  │   └── styles.xml      ← 样式
  ├── _rels/
  └── [Content_Types].xml
```

所以这个项目处理 docx 的方式就是：**解压 → 改 XML → 重新打包**。

<a id="s0-2"></a>
### 0.2 XML 是什么

XML 是一种用标签（tag）来标记文字的语言，和 HTML 很像。Word 文档里的文字是用这样的方式存的：

```xml
<w:p>                          <!-- 这是一个段落（paragraph） -->
  <w:r>                        <!-- 这是一个 run（一段连续的文字） -->
    <w:rPr>                    <!-- run 的属性（字体、大小、颜色等） -->
      <w:b/>                   <!-- 加粗 -->
      <w:sz w:val="24"/>       <!-- 字号 24 -->
    </w:rPr>
    <w:t>北流市</w:t>          <!-- 文字内容 -->
  </w:r>
</w:p>
```

**关键概念：**
- `<w:p>` = paragraph（段落）
- `<w:r>` = run（一段具有相同格式的文字，可以理解成"格式块"）
- `<w:t>` = text（实际的文字内容）
- `<w:rPr>` = run properties（run 的属性：是否加粗、什么字体、多大字号等）

<a id="s0-3"></a>
### 0.3 什么是"run 被拆碎"？

Word/WPS 在保存文件时，经常把一句完整的话切成很多小段：

本来应该是：`<w:t>北流市</w:t>`

实际上可能是：
```
<w:r><w:t>北</w:t></w:r>
<w:r><w:t>流</w:t></w:r>
<w:r><w:t>市</w:t></w:r>
```

一个字一个 run！这样你在单个 run 里找"北流市"是肯定找不到的。  
这就是 `merge_runs.py` 要解决的问题。

## 2. `src/core/merge_runs.py`——把 Word 里"碎掉的文字"拼起来

<a id="s2-1"></a>
### 2.1 这个文件要解决什么问题

**问题**：Word/WPS 保存文件时，经常一句话被切成几十段，比如：

```
<w:r><w:t>广西电</w:t></w:r>        ← 一个 run
<w:r><w:t>网有限责</w:t></w:r>      ← 又一个 run
<w:r><w:t>任公司</w:t></w:r>         ← 又一个 run
```

如果在每个 run 里单独搜索"广西电网有限责任公司"——**永远搜不到**，因为没有一个 run 包含了完整的"广西电网有限责任公司"。

**解决方案**：把相邻的、格式相同的 run 合并成一个。

<a id="s2-2"></a>
### 2.2 代码逐块讲解（按文件顺序）

#### 第 1-3 行：文件编码声明和说明

```python
# -*- coding: utf-8 -*-
"""
段落级 run 合并：把 Word XML 中相邻且格式相同的 <w:r> 合并为一个,
消除 Word 随机拆分 run 导致的文本断裂问题。
"""
```

- `# -*- coding: utf-8 -*-`：告诉 Python 这个文件用 UTF-8 编码（支持中文）
- 三引号 `"""..."""`：多行注释，描述这个文件是做什么的

#### 第 13-17 行：导入工具和定义常量

```python
from copy import deepcopy
from lxml import etree

NS = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}
W = 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'
```

- `from copy import deepcopy`：导入"深复制"功能。`deepcopy(x)` 会创建一个和 x 完全一样的副本，修改副本不影响原版。
- `from lxml import etree`：导入 lxml 库（专门处理 XML 的第三方库）。`etree` 是这个库的核心模块，可以读/写/搜索 XML。
  - `etree.parse(文件)` — 读取一个 XML 文件
  - `tree.iter(标签名)` — 在 XML 树中找到所有指定标签的元素
  - `element.findall(标签名)` — 找到一个元素下的所有子元素
  - `element.find(标签名)` — 找到一个元素下的第一个子元素
- `W = 'http://...'`：这是 Word XML 的**命名空间**。简单理解：Word 的 XML 标签前面都带这个长长的网址来标识"这是 Word 的东西"。`_tag('p')` 就拼出完整的标签名 `{http://...}p`。

#### 第 20-21 行：辅助函数——拼标签名

```python
def _tag(local):
    return f'{{{W}}}{local}'
```

- `def`：定义一个函数
- `_tag`：函数名。前面加下划线表示"这是内部函数，不对外公开"
- `local`：参数（你想拼的标签名，如 `'p'`、`'r'`）
- `f'{{{W}}}{local}'`：f-string，Python 的字符串模板。`{W}` 是命名空间网址，`{local}` 是标签名
- 效果：`_tag('p')` → `'{http://...}p'`

**为什么要这样写？** Word 的 XML 标签名很长（`{http://schemas...}p`），每次都手写太麻烦、容易写错。用一个函数自动拼，方便又不容易出错。

#### 第 24-26 行：忽略的格式属性

```python
IGNORED_RPR_TAGS = frozenset([
    _tag('spacing'),   # 字间距微调，WPS/Word 排版自动生成
    _tag('kern'),      # 字距调整
])
```

- `frozenset(...)`：一个"不可变的集合"，效率比列表高，内容不能修改
- `spacing`：字间距（WPS 排版自动加的）
- `kern`：字距调整
- 这两个属性是 Word/WPS 自动加的**排版微调**，不影响文字的语义。两个 run 的字体、大小一样但间距不同，应该合并。

#### 第 30-41 行：比较两个 run 的格式是否相同

```python
def _normalize_rpr(rpr_elem):
    """把 <w:rPr> 序列化为可比较的规范字符串。
    忽略排版微调属性（spacing/kern），这些不影响语义格式。"""
    if rpr_elem is None:
        return b''
    children = [c for c in rpr_elem if c.tag not in IGNORED_RPR_TAGS]
    if not children:
        return b''
    norm = etree.Element(_tag('rPr'))
    for child in sorted(children, key=lambda e: e.tag):
        norm.append(deepcopy(child))
    return etree.tostring(norm, method='c14n2')
```

这段代码就是把一个 run 的"格式属性"转成一段字符串，方便比较两个 run 的格式是否一样。

逐句讲解：

- `def _normalize_rpr(rpr_elem):` — 定义一个函数，参数 `rpr_elem` 是一个 `<w:rPr>` 元素（run 的属性）
- `if rpr_elem is None:` — 如果这个 run 没有任何格式属性（为空）
- `return b''` — 返回空字节串（`b''` 是 bytes 类型的空值，不是普通的字符串 `''`）

- `children = [c for c in rpr_elem if c.tag not in IGNORED_RPR_TAGS]`
  - 这是**列表推导式**（list comprehension），是 Python 的一种简洁写法
  - 等价于：
    ```python
    children = []
    for c in rpr_elem:
        if c.tag not in IGNORED_RPR_TAGS:
            children.append(c)
    ```
  - 意思：遍历 `rpr_elem` 的所有子元素，只保留那些**不在忽略列表**里的（即不保留 spacing 和 kern）

- `if not children:` — 如果过滤后没有子元素了
- `return b''` — 也返回空字节串

- `norm = etree.Element(_tag('rPr'))` — 创建一个新的 `<w:rPr>` 空标签
- `for child in sorted(children, key=lambda e: e.tag):` — 把子元素**按标签名字排序**（保证顺序一致）
  - `sorted(列表, key=函数)`：按 key 函数返回的值排序
  - `lambda e: e.tag`：lambda 是"匿名函数"，意思是"对每个元素 e，取它的 tag（标签名）"
- `norm.append(deepcopy(child))` — 把每个子元素的**副本**添加到 norm 中
- `return etree.tostring(norm, method='c14n2')` — 把 norm 转成字节串返回
  - `c14n2` 是一种标准化格式，保证属性顺序固定，方便比较

**为什么要这样做？** 两个 run 的格式是否相同，不能直接对比 `<w:rPr>` 标签，因为：
1. 属性的顺序可能不同
2. 有些属性（spacing/kern）不需要考虑
所以需要"标准化"之后再比较。

#### 第 44-49 行：获取 run 中的文字

```python
def _get_run_text(run):
    """拼接 run 内所有 <w:t> 的文本。"""
    parts = []
    for t in run.findall(_tag('t')):
        parts.append(t.text or '')
    return ''.join(parts)
```

- `run.findall(_tag('t'))`：在 run 中找到所有 `<w:t>` 标签（一个 run 可能有多个 `<w:t>`）
- `t.text or ''`：取 `<w:t>` 标签里面的文字。用 `or ''` 意思是：如果是 None（空），就当作空字符串
- `parts.append(...)`：把文字添加到 parts 列表末尾
- `''.join(parts)`：把 parts 列表中的所有字符串**拼接**成一个字符串
  - 例：`''.join(['北', '流', '市'])` → `'北流市'`

#### 第 52-65 行：把文字写回 run

```python
def _set_run_text(run, text):
    """把文本写入 run 的单个 <w:t>，删除多余的 <w:t>。"""
    t_elems = run.findall(_tag('t'))
    if not t_elems:
        t_elem = etree.SubElement(run, _tag('t'))
        t_elems = [t_elem]

    t_elems[0].text = text
    if text and (text[0] == ' ' or text[-1] == ' '):
        t_elems[0].set('{http://www.w3.org/XML/1998/namespace}space', 'preserve')

    for extra in t_elems[1:]:
        run.remove(extra)
```

- `t_elems = run.findall(_tag('t'))`：找到 run 里所有的 `<w:t>` 元素
- `if not t_elems:`：如果 run 里没有 `<w:t>`（空的）
- `t_elem = etree.SubElement(run, _tag('t'))`：在 run 里**新建**一个 `<w:t>`
  - 第一个参数是父元素，第二个参数是标签名

- `t_elems[0].text = text`：把文字写入**第一个** `<w:t>`
  - `列表[0]` 是列表的第一个元素

- `if text and (text[0] == ' ' or text[-1] == ' '):`：如果文字**首尾有空格**
  - `text[0]`：第一个字符
  - `text[-1]`：最后一个字符
  - Word 默认会吃掉 `<w:t>` 首尾的空格，所以需要告诉它"保留空格"
  - `t_elems[0].set('{http://www.w3.org/XML/1998/namespace}space', 'preserve')`：设置 `xml:space="preserve"` 属性，意思是"保留空格"

- `for extra in t_elems[1:]:`：从第二个 `<w:t>` 开始遍历（`[1:]` 表示"从下标 1 开始到结束"）
- `run.remove(extra)`：删除多余的 `<w:t>`，只保留第一个

#### 第 68-76 行：判断 run 是否"纯净"

```python
def _is_pure_text_run(run):
    """判断 run 是否只包含 rPr 和 t 元素（纯文本 run）。
    含图片、换行符、制表符等特殊元素的 run 不参与合并。"""
    for child in run:
        tag = child.tag
        if tag == _tag('rPr') or tag == _tag('t'):
            continue
        return False
    return True
```

- `for child in run:`：遍历 run 的所有子元素
- 如果子元素是 `rPr`（格式属性）或 `t`（文本内容），说明是正常的文本 run → `continue`（跳过，继续检查下一个）
- 如果**不是**这两种之一（比如是图片 `<w:drawing>`、换行 `<w:br>`、制表符 `<w:tab>`），说明这个 run 包含特殊内容 → `return False`（不参与合并）
- 全部检查完都没问题 → `return True`

**为什么不能合并有图片的 run？** 图片、换行符这些特殊元素有自己独立的结构，合并到文本 run 里会破坏文档显示。

#### 第 79-110 行：核心——合并一个段落内的 run

```python
def merge_runs_in_paragraph(para):
    """合并单个段落内相邻格式相同的 run。返回合并的 run 对数。"""
    merged_count = 0
    children = list(para)
    run_tag = _tag('r')

    i = 0
    while i < len(children) - 1:
        current = children[i]
        if current.tag != run_tag or not _is_pure_text_run(current):
            i += 1
            continue

        nxt = children[i + 1]
        if nxt.tag != run_tag or not _is_pure_text_run(nxt):
            i += 1
            continue

        cur_rpr = current.find(_tag('rPr'))
        nxt_rpr = nxt.find(_tag('rPr'))

        if _normalize_rpr(cur_rpr) == _normalize_rpr(nxt_rpr):
            merged_text = _get_run_text(current) + _get_run_text(nxt)
            _set_run_text(current, merged_text)
            para.remove(nxt)
            children = list(para)
            merged_count += 1
        else:
            i += 1

    return merged_count
```

这段是**核心逻辑**，我们一点一点拆：

- `merged_count = 0`：记录合并了多少对 run
- `children = list(para)`：把段落的所有子元素转成列表（用 `list()` 是因为后面会删除元素，直接用 `para` 的原始列表会出问题）
- `run_tag = _tag('r')`：拼出完整的 run 标签名

- `i = 0`：从第 0 个开始检查
- `while i < len(children) - 1:`：循环，条件是 `i` 小于列表长度减 1（因为每次要比较 `current` 和 `nxt`，不能越界）
  - `len(children)` → 列表长度（有几个元素）

- `current = children[i]`：当前元素
- `if current.tag != run_tag or not _is_pure_text_run(current):`：如果当前元素**不是 run** 或者**不是纯文本 run**
- `i += 1; continue`：跳过，检查下一个

- `nxt = children[i + 1]`：下一个元素
- 同样检查：不是 run 或不是纯文本就跳过

- `cur_rpr = current.find(_tag('rPr'))`：找当前 run 的格式属性
- `nxt_rpr = nxt.find(_tag('rPr'))`：找下一个 run 的格式属性
- `if _normalize_rpr(cur_rpr) == _normalize_rpr(nxt_rpr):`：**格式相同**才合并
  - 如果格式相同：
    - `merged_text = _get_run_text(current) + _get_run_text(nxt)`：把两个 run 的文字拼起来
    - `_set_run_text(current, merged_text)`：拼好的文字写回当前 run
    - `para.remove(nxt)`：**删除**下一个 run（合并到当前 run 了）
    - `children = list(para)`：**重新获取**子元素列表（因为删了一个元素，列表变了）
    - `merged_count += 1`：合并计数加 1
    - **不递增 i**：继续检查当前 run 和再下一个 run（可能三个以上连续格式相同）
  - 如果格式不同：
    - `i += 1`：跳过，检查下一对

- `return merged_count`：返回合并了多少对

**为什么要用 while 而不是 for？** 因为合并后列表变了（删除了元素），`for i in range(len(children))` 在这种情况下会出问题。while 让我们可以灵活控制 `i` 的增减。

```python
def build_replacements(schema):
    """从 schema.json 构建替换规则列表。"""
    text_rules = []
    regex_rules = []
    seen_searches = set()

    for var in schema['variables']:
        var_id = var['var_id']
        vtype = var['type']

        if vtype == 'atomic':
            _add_atomic_rule(var_id, var.get('value'), text_rules, regex_rules, seen_searches)
            for i, sv in enumerate(var.get('search_variants', []), 1):
                key = f"{var_id}_v{i}"
                _add_atomic_rule(key, sv, text_rules, regex_rules, seen_searches)

        elif vtype == 'composite':
            for sv in var.get('sub_values', []):
                key = f"{var_id}_sub"
                _add_atomic_rule(key, sv, text_rules, regex_rules, seen_searches)
            for av in var.get('atomic_values', []):
                _add_atomic_rule(av['key'], av['value'], text_rules, regex_rules, seen_searches)

        elif vtype == 'paragraph':
            for av in var.get('atomic_values', []):
                _add_atomic_rule(av['key'], av['value'], text_rules, regex_rules, seen_searches)

    text_rules.sort(key=lambda r: len(r[0]), reverse=True)

    return text_rules, regex_rules
```

- `text_rules = []`：用于存"文本替换规则"的列表。每条规则是 `(搜索值, 替换值, 变量名)` 这样的元组
- `regex_rules = []`：用于存"正则替换规则"的列表
- `seen_searches = set()`：记录已经见过的搜索值，防止重复添加
  - `set()`（集合）类似列表，但每个元素只能出现一次

- `for var in schema['variables']:`：遍历 schema.json 中的每个变量
  - `schema` 是一个读进来的 Python 字典
  - `schema['variables']` 是变量列表

- **atomic 类型**：直接添加主值，再添加所有搜索变体（`search_variants`）
  - `enumerate(列表, 1)`：遍历列表，从 1 开始计数
  - 比如 `search_variants=["建陶储能", "建陶站"]` → 生成 key `C_v1`, `C_v2`

- `text_rules.sort(key=lambda r: len(r[0]), reverse=True)`：按搜索值的**长度从大到小排序**

```python
def _add_atomic_rule(key, value, text_rules, regex_rules, seen_searches):
    """添加单个原子值的替换规则。"""
    if not value or not value.strip():
        return

    if key in _SPACE_NORMALIZED:
        rule = _SPACE_NORMALIZED[key]
        regex_rules.append((rule['pattern'], rule['replacement'], key))
        return

    if key in _CONTEXT_ANCHORS:
        search, replace = _CONTEXT_ANCHORS[key]
        if search not in seen_searches:
            seen_searches.add(search)
            text_rules.append((search, replace, key))
        return

    if _is_short_numeric(value):
        return

    placeholder = '{{' + key + '}}'
    if value not in seen_searches:
        seen_searches.add(value)
        text_rules.append((value, placeholder, key))
```

- `if not value or not value.strip():`：如果值为空或全是空格 → 啥也不做
  - `value.strip()`：去掉首尾空格

- 注册空格归一化规则：`regex_rules` 添加 `(编译好的正则, 替换值, 变量名)`
- 注册上下文锚定规则：`text_rules` 添加 `(搜索文本, 替换文本, 变量名)`
- 如果是短数值但没有预定义的锚定 → `return`（跳过，不自动替换，防止误伤）
- 普通值：构造 `{{变量名}}` 占位符，添加到 text_rules

**为什么要有 `seen_searches`？** 多个变量可能对应同一个搜索值。比如 C 和它的 search_variants 可能有重复。用集合记录已经添加的搜索值，避免重复添加相同的替换规则。

```python
def _has_cross_run_match(t_elements, full_text, text_rules, regex_rules):
    """检查是否存在跨 run 的匹配"""
    single_texts = set()
    for _, text in t_elements:
        if text:
            single_texts.add(text)

    for pattern, replacement, key in regex_rules:
        if pattern.search(full_text):
            if not any(pattern.search(t) for t in single_texts if t):
                return True

    for search, replacement, key in text_rules:
        if search in full_text:
            if not any(search in t for t in single_texts if t):
                return True

    return False
```

- `single_texts`：收集每个 `<w:t>` 的独立文本
- 遍历每条规则：
  - 检查"整段拼接后"是否匹配
  - 检查"每个单独 `<w:t>` 中"是否也匹配
  - 如果拼接后匹配但单独 `<w:t>` 不匹配 → 说明这个匹配是**跨 run**的 → 返回 True

**为什么要有这个检查？** 段落拼接替换会破坏格式（所有文字被放到第一个 `<w:t>` 里），所以只在**必须拼接**的情况下才做。如果一个段落的每个 `<w:t>` 自身就能匹配，就不需要拼接了。

```python
def replace_in_tree(tree, text_rules, regex_rules):
    total_count = 0
    affected_vars = set()

    # 第一轮：段落级拼接替换（仅跨 run 匹配的段落）
    for para in tree.iter(_tag('p')):
        t_elements = _collect_t_elements(para)
        if not t_elements:
            continue

        old_full = ''.join(t.text or '' for t, _ in t_elements)
        if not old_full.strip():
            continue

        if not _has_cross_run_match(t_elements, old_full, text_rules, regex_rules):
            continue

        new_full, count, affected = _apply_rules(old_full, text_rules, regex_rules)
        if count > 0:
            total_count += count
            affected_vars.update(affected)
            _redistribute_text(t_elements, new_full)

    # 第二轮：单 <w:t> 级替换
    for t_elem in tree.iter(_tag('t')):
        text = t_elem.text
        if not text:
            continue

        new_text, count, affected = _apply_rules(text, text_rules, regex_rules)
        if count > 0:
            total_count += count
            affected_vars.update(affected)
            t_elem.text = new_text
            if '{{' in new_text or (new_text and (new_text[0] == ' ' or new_text[-1] == ' ')):
                t_elem.set('{http://www.w3.org/XML/1998/namespace}space', 'preserve')

    return total_count, affected_vars
```

**第一轮：段落级别**
- 遍历每个段落
- 如果段落有跨 run 匹配，就拼接整个段落的文本，应用替换规则
- 替换结果写回第一个 `<w:t>`，其余清空

**第二轮：单 `<w:t>` 级别**
- 遍历每个 `<w:t>`
- 在单个 `<w:t>` 内部做替换
- 如果替换结果含有 `{{...}}` 或者首尾有空格，设置 `xml:space="preserve"`

**为什么需要两轮？**
- 第一轮处理跨 run 的情况（要拼接文字），但会破坏格式
- 第二轮处理单 run 的情况（不需要拼接），不破坏格式
- 第一轮处理过的 `<w:t>` 里的内容已被合并，第二轮不会重复处理（因为那些 run 内已经找不到可匹配的原始文本了）

```python
def _print_coverage_report(schema, all_affected_vars):
    """打印变量覆盖报告。"""
    all_var_keys = set()
    for var in schema['variables']:
        var_id = var['var_id']
        vtype = var['type']
        if vtype == 'atomic' and var.get('value'):
            all_var_keys.add(var_id)
            for i in range(len(var.get('search_variants', []))):
                all_var_keys.add(f"{var_id}_v{i+1}")
        elif vtype == 'composite':
            for av in var.get('atomic_values', []):
                all_var_keys.add(av['key'])
            for sv in var.get('sub_values', []):
                all_var_keys.add(f"{var_id}_sub")
        elif vtype == 'paragraph':
            for av in var.get('atomic_values', []):
                all_var_keys.add(av['key'])

    missed = all_var_keys - all_affected_vars
    if missed:
        print(f'\n未命中的变量 ({len(missed)}):')
        for k in sorted(missed):
            print(f'  - {k}')
    else:
        print(f'\n所有变量均已覆盖!')
```

- `all_var_keys`：收集**理论上应该被替换**的所有变量 key
- `all_affected_vars`：实际被替换的变量 key（从 `replace_in_tree` 返回）
- `all_var_keys - all_affected_vars`：**差集**运算——应该在但没出现的，就是"未命中"
- 输出未命中的变量列表

**为什么要有这个报告？** 76 个变量不能用人眼一个个核对。自动输出"哪些没替换到"，快速发现 schema 定义问题或文档版本不匹配。

## 5. `variables/schema.json`——变量字典

<a id="s5-1"></a>
### 5.1 这个文件是干什么的

这是一个 JSON 文件（JavaScript Object Notation，一种通用的数据格式），里面记录了从完整报告中提取出来的**所有可变内容**。

可以把它想象成一张表：

| 变量 ID | 原始值 | 类型 | 分类 | 说明 |
|---------|--------|------|------|------|
| A | 北流市 | atomic | 地区 | 县级市名称 |
| A1 | 玉林市 | atomic | 地区 | 地级市名称 |
| A2 | 北海 | atomic | 地区 | 相邻地区名 |
| D18 | 300MW | atomic | 项目 | 项目额定功率 |
| C4 | 横山风电项目（100MW）…… | paragraph | 项目列表 | 一长串风电项目 |
| ... | ... | ... | ... | ... |

JSON 格式在 Python 里会自动变成**字典**（dict）和**列表**（list），可以直接用。

<a id="s5-2"></a>
### 5.2 JSON 结构解析

```json
{
  "meta": { ... },           ← 元信息
  "variables": [ ... ]       ← 变量列表（核心）
}
```

整体是一个**对象**（Python 里叫 dict），有两个字段：
- `meta`：数据来源、变量总数、说明
- `variables`：一个数组（Python 里叫 list），里面每个元素是一个变量定义

<a id="s5-3"></a>
### 5.3 三种变量类型

**类型 1：atomic（原子值）**
```json
{
  "var_id": "A",
  "value": "北流市",
  "type": "atomic",
  "category": "地区",
  "description": "县级市名称"
}
```
- 最简单的变量，一个 key 对应一个 value
- 替换时直接在文档中搜索"北流市"，替换成 `{{A}}`

**类型 2：composite（复合值）**
```json
{
  "var_id": "D2",
  "value": null,
  "type": "composite",
  "category": "电网数据",
  "atomic_values": [
    {"key": "D2_500kv_count", "value": "2 座 500kV 公用变电站", "context": "玉林"},
    {"key": "D2_500kv_capacity", "value": "3500MVA", "context": "500kV总容量"},
    ...
  ]
}
```
- 一个逻辑变量包含多个**分散在文档不同位置**的值
- 每个子值用自己的 key 替换（如 `{{D2_500kv_count}}`、`{{D2_500kv_capacity}}`）

**类型 3：paragraph（段落）**
```json
{
  "var_id": "D23",
  "value": null,
  "type": "paragraph",
  "category": "潮流",
  "description": "方案潮流数据（多段）",
  "raw_text": "方案一各方式下建陶～建陶 II 线最大运行潮流区间约13MVA~184MVA..."
}
```
- 整段文字，上下文太复杂，**不自动替换**
- `raw_text` 只是人工参考

<a id="s5-4"></a>
### 5.4 为什么变量分三种

| 类型 | 自动替换？ | 为什么 |
|------|-----------|--------|
| **atomic** | ✅ 全自动 | 简单值，"北流市"就是"北流市"，在哪都一样 |
| **composite** | ✅ 半自动 | 每个子值也是确定的，可以自动替换 |
| **paragraph** | ❌ 需人工 | 一长段文字，不同项目版本的内容完全不同，自动替换必出错 |

<a id="s5-5"></a>
### 5.5 search_variants（搜索变体）是干什么的

变量 C 的主值是"建陶"（2 个字），但它在文档中可能出现为"建陶储能""建陶站""北流市建陶"等形式。如果只搜"建陶"替换成 `{{C}}`：
- "建陶储能" → `{{C}}储能`（正确）
- "建陶站" → `{{C}}站`（正确）
- "北流市建陶" → `北流市{{C}}`（正确？但等一下，"北流市"是变量 A，`{{A}}{{C}}` 看起来不对）

但是如果"建陶站"和"建陶储能"也需要分别作为变量替换，那搜索"建陶"就会误伤。所以 `search_variants` 提供了更长的、更具体的搜索版本。

<a id="s5-6"></a>
### 5.6 occurrences（多处出现）是什么

变量 E（线路）在文档不同段落中有不同的值：
- 主干线路段："南宁～A1，逢宜～A1……"
- 故障线路段："美陆 Ⅰ 线故障……"
- 新建线路段："新建容县二～平坡……"

每个上下文里，E 的取值都不一样。`occurrences` 数组记录了这些不同的取值。

```python
def extract_text_from_tree(tree):
    texts = tree.xpath('//w:t/text()', namespaces=NS)
    return ''.join(texts)
```

- **XPath 查询**：`//w:t/text()` 的意思
  - `//`：不管在什么层级
  - `w:t`：找到所有 `<w:t>` 标签
  - `/text()`：提取标签内的文字
- 返回所有 `<w:t>` 文字拼接成的完整字符串
- `namespaces=NS`：告诉 XPath `w:` 前缀对应哪个命名空间

```python
def extract_run_texts_from_tree(tree):
    runs = tree.xpath('//w:r', namespaces=NS)
    run_texts = []
    for r in runs:
        t_elems = r.findall('.//w:t', namespaces=NS)
        run_text = ''.join(t.text or '' for t in t_elems)
        if run_text:
            run_texts.append(run_text)
    return run_texts
```

- 和上一个函数的区别：
  - `extract_text_from_tree`：返回**一个**完整的字符串（所有文字拼在一起）
  - `extract_run_texts_from_tree`：返回**一个列表**，每个元素是一个 run 里的文字
- 这个列表用于检测"跨 run 断裂"——如果一个值要跨两个 run 才能找到，就是断裂的

```python
def count_elements(tree):
    paragraphs = len(tree.xpath('//w:p', namespaces=NS))
    tables = len(tree.xpath('//w:tbl', namespaces=NS))
    sections = len(tree.xpath('//w:sectPr', namespaces=NS))
    runs = len(tree.xpath('//w:r', namespaces=NS))
    return paragraphs, tables, sections, runs
```

- 返回四个统计值：段落数、表格数、分节符数、run 数
- `len(xpath结果)`：xpath 返回列表，`len()` 统计数量

```python
def check_cross_run_breaks(run_texts, value):
    single_run_count = sum(1 for rt in run_texts if value in rt)
    joined = ''.join(run_texts)
    total_count = joined.count(value)
    broken_count = total_count - single_run_count
    return total_count, single_run_count, broken_count
```

检测一个值是否有"跨 run 断裂"：
- `sum(1 for rt in run_texts if value in rt)`：在**单个 run 内部**就能匹配到的次数
  - `条件表达式`：对每个 rt，检查 value 是否在 rt 里
  - `sum(1 for ... if ...)`：计数满足条件的元素个数
- `joined = ''.join(run_texts)`：把所有 run 的文字拼起来
- `total_count = joined.count(value)`：在**完整文本**中出现的总次数
- `broken_count = total_count - single_run_count`：总次数 - 单 run 内匹配次数 = 跨 run 断裂次数

**例子：**
```
run_texts = ["广西电", "网有限责", "任公司"]
value = "广西电网有限责任公司"

single_run_count = 0（没有一个 run 完整包含"广西电网有限责任公司"）
total_count = 1（完整拼接后出现了 1 次）
broken_count = 1（断裂了 1 次）
```


**双重循环**检测包含关系：

- `i != j`：不是同一个变量
- `vj in vi and len(vj) < len(vi)`：vj 是 vi 的一部分，且 vj 更短

```python
    """将模板 docx 中的 {{key}} 替换为用户提供的值"""
    output_path = UPLOAD_DIR / f'output_{int(time.time())}.docx'

    with tempfile.TemporaryDirectory() as tmpdir:
        # 解压
        with zipfile.ZipFile(docx_path) as zin:
            zin.extractall(tmpdir)
            namelist = zin.namelist()

        xml_parts = _find_xml_parts(namelist)

        for part in xml_parts:
            part_path = os.path.join(tmpdir, part)
            tree = etree.parse(part_path)

            # 第一轮：替换 {{key}} 占位符
            # 遍历 XML 中每个 <w:t> 文本节点
            # 对每个 <w:t>，检查是否包含 {{A}}、{{D18}} 等
            # 有则替换为用户填的新值
            for t_elem in tree.iter(f'...}}t'):
                if not t_elem.text:
                    continue
                for key, new_val in values.items():
                    pattern = '{{' + key + '}}'
                    if pattern in t_elem.text:
                        t_elem.text = t_elem.text.replace(pattern, str(new_val))

            # 第二轮：对于没有 {{var_id}} 占位符的字段
            # 直接在文档中搜索 schema 里的原始文本
            # 找到就替换（不一定成功，因文本可能不精确匹配）
            if schema_originals:
                for key, new_val in values.items():
                    orig = schema_originals.get(key, '')
                    if orig and orig != str(new_val):
                        for t_elem in tree.iter(f'...}}t'):
                            if t_elem.text and orig in t_elem.text:
                                t_elem.text = t_elem.text.replace(orig, str(new_val))
        ...
```

```python
def scan_placeholders(docx_path):
    """从模板 docx 中扫描所有 {{key}} 占位符"""
    keys = set()
    with zipfile.ZipFile(docx_path) as z:
        parts = ['word/document.xml']
        for name in z.namelist():
            if re.match(r'word/header\d+\.xml$', name):
                parts.append(name)
        for part in parts:
            tree = etree.fromstring(z.read(part))
            for t in tree.iter(f'...}}t'):
                if t.text and '{{' in t.text:
                    for m in re.finditer(r'\{\{([^}]+)\}\}', t.text):
                        keys.add(m.group(1))
    return keys
```

- 直接打开 ZIP 包中的 XML 文件（不解压到磁盘）
- 遍历所有 `<w:t>` 节点
- 用正则 `\{\{([^}]+)\}\}` 提取 `{{XXX}}` 中的 `XXX`
- 返回一个集合（set），供后续匹配变量定义

```python
    uvicorn.run('src.api.main:app', host=args.host, port=args.port, reload=True)
```

- `reload=True`：代码修改后自动重启（开发时方便）
- `host='0.0.0.0'`：监听所有网络接口，局域网内其他设备也能访问
